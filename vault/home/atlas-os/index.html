What it is

Four systems. One desktop app.

Most tools do one thing. Atlas OS integrates a knowledge base, an AI development environment, an infinite canvas workspace, and a native web browser — all backed by local models with no cloud dependency.

🗄

Knowledge Vault

Pages live at vault/{category}/{slug}/index.html. AI agents write and index them automatically. Keyword TF-IDF + 384-dim semantic search — fully offline.

🤖

AI IDE

Five specialized agents, CodeMirror 6 editor, PTY terminal, Mermaid preview, voice input, sub-agents, and every tool an AI coder needs — all running on your local models.

⬡

Infinite Canvas

Pan/zoom workspace with draggable panels (terminal, editor, chat, browser, sub-canvas). Connect panels with edges. Build agent workflows. Save as blueprints.

🌐

Real Browser

Two tab kinds: Vault iframe for local pages and Web native child WebView. Browse Google, GitHub, YouTube. No X-Frame-Options limits.

Knowledge Vault

A file system that thinks.

The vault is a folder of plain HTML files. The indexer understands them; the AI agents extend them; the backlink panel connects them. Every page is readable in any browser even without Atlas OS installed.

vault/{category}/{slug}/index.html ← source of truth
          │
tools/indexer.py ← two-pass: extract → resolve backlinks
tools/embedder.py ← 384-dim vectors (Ollama or sentence-transformers)
          │
.index/pages.json ← machine-readable (API + CLI + IDE)
.index/pages.js ← browser-loadable (window.ATLAS_INDEX)
.index/embeddings.json ← cosine similarity vectors
          │
api/server.py port 4242 ← REST: /api/search · /api/semantic · /api/pages
cli/atlas.py ← terminal: search · list · open · chat
ide/ ← Tauri IDE reads .index directly (offline keyword)
ui/index.html ← standalone browser search (Fuse.js, no server needed)

Search modes

  • Keyword — TF-IDF; title(3) > headings/desc(2) > body(1)
  • Semantic — 384-dim cosine similarity (requires embedder.py)
  • Hybrid — IDE combines both results live
  • Scope filter: per-category or cross-vault
  • Archive exclusion: include_archive param

Auto re-index triggers

  • After any vault_write tool call by an agent
  • File watcher (5s debounce, Rust notify crate)
  • Emits vault:reindexed event — UI auto-refreshes
  • BacklinkPanel, VaultHome, Graph View all respond to this event
  • Manual: python tools/indexer.py
Backlinks & Similar Notes New
🔗 Backlink panel

In any Vault tab, click the ⟳ (Link) button in the address bar to open a right-side panel showing every page that links to the current one.

≈ Similar notes

The same panel shows semantically similar pages via cosine similarity on embeddings. Requires embedder.py to have been run at least once.

⚡ Live refresh

Listens for vault:reindexed and updates backlinks and similar notes automatically whenever the vault changes.

🗺 Knowledge graph

Press Ctrl+Shift+G or click the graph icon in the header. D3 force-directed graph of all vault pages and their links. Click a node to open the page.

AI Agents

Five specialists, not one giant prompt.

Each agent has a focused system prompt, the right subset of tools, and its own vault office (a folder of notes it reads at startup). Switch agents mid-conversation from the AgentSwitcher. Sub-agents (explore + general) run in parallel for research-heavy tasks.

Vault
builtin:vault

Default. Searches your vault before answering. Cites local pages. Saves new knowledge. Best for research and Q&A grounded in your own notes.

vault_search vault_write read_file web_search web_fetch
Atlas-Maker
builtin:atlas-maker

Writes new vault HTML pages from any conversation, voice note, or research result. Shows a confirmation card before saving. Re-indexes on write.

vault_write vault_search web_fetch subagent
Coder
builtin:coder

Reads, edits, and creates source files. Runs shell commands. Multi-file edits, grep, glob, terminal output. Read-before-edit enforced.

read_file edit_file multi_edit bash_run grep glob
Sentor
builtin:sentor

Interfaces with a local Flowise instance (port 3000). Lists and runs visual agent flows. Requires Flowise running in your Sentor path.

sentor_list_flows sentor_run_flow vault_search
Orkestra
builtin:orkestra

Coordinates other agents. Uses agent_invoke to delegate tasks to Vault, Coder, or Atlas-Maker as sub-steps of a larger goal.

agent_invoke canvas_read_state blueprint_save subagent
Agent Offices New

Each agent has a personal office at vault/agents/{slug}/ — a collection of vault pages it reads at startup (Self-RAG). The office holds the agent's memory, meeting notes, and decisions. Open with Ctrl+Shift+A.

🏢 Agent offices

4 built-in offices: vault, atlas-maker, coder, sentor. Each agent reads its own office pages via vault_self_context before responding.

📋 /decision command

Use /decision {text} to log a decision to the active agent's office log. Persists across sessions.

📝 /meeting command

Use /meeting {topic} to create a timestamped meeting note in the active agent's office.

🔄 Self-RAG

Agents inject their own office context into every prompt via vault_self_context. They remember what they decided and wrote in past sessions.

Local AI Providers
LM Studio

Point at localhost:1234. Any loaded model works. Set the exact model ID in Settings → Models (e.g. google/gemma-4-e4b).

Ollama

Point at localhost:11434. Pull any model: ollama pull <name>. Semantic embeddings: ollama pull all-minilm.

Cloud providers

OpenAI, Anthropic, Google, xAI, Cerebras, Groq, DeepSeek — add API key to OS keyring. No key stored on disk.

Infinite Canvas

A spatial workspace for your tools.

The canvas is a pannable, zoomable infinite plane where you can place any IDE panel — terminal, editor, chat, browser, or a sub-canvas. Connect panels with wires. Arrange, save, and recall layouts as blueprints.

Panel types
type: terminal

Terminal

Full PTY terminal inside a canvas panel. Set cwd and initCmd in meta to auto-run a command on spawn.

type: editor

Editor

CodeMirror 6 editor panel. Opens any file from the vault or workspace.

type: chat

Chat

AI chat panel with its own isolated session ID. Multiple chat panels don't share history.

type: web

Web

Native WebView inside the canvas. Browse any URL. Zooms and pans with the canvas.

type: vault-home

Vault Home

Inline vault search panel. Search your entire vault without leaving the canvas.

type: canvas

Sub-Canvas

A nested canvas inside the main canvas. Has its own viewport and children. Export contents to vault as a blueprint page.

type: agent

Agent

Created via right-click → Agent. Configure name, tools, system prompt. Save as custom agent.

type: preview

Preview

File preview panel. Renders HTML, images, Markdown with Mermaid support.

Canvas UX New
⬛ Fullscreen panel

Double-click any panel's title bar → the panel goes fullscreen (fixed inset-0). Press Esc to exit. Double-clicking the title text still opens rename.

− Minimize to dock

Click the − button on a panel → it disappears from the canvas and appears as a chip in the dock strip at the bottom. Click the chip to restore. × to close.

📌 Pin to screen

Click the pin icon → panel attaches to screen coordinates (not canvas). Stays in place when you pan or zoom. Useful for always-visible terminals.

↑ Z-index capture

Click any panel — including pinned ones — and it comes to front. Uses onPointerDownCapture so title-bar button clicks don't block the bring-to-front.

🔗 Connection edges

Drag from a panel port to another panel to draw a connection edge. Edges represent data flow in agent workflows.

💾 Persistent state

All panels, positions, connections, minimize state, and viewport are saved to atlas-canvas.json (Tauri LazyStore) and restored on next launch.

Agent Builder & Blueprint
🛠 Agent Builder

Right-click on canvas → Add Agent panel. Configure name, tool subset, and system prompt. Save → a real agent is registered and a Chat panel opens automatically.

🗺 Blueprint

AI can call blueprint_save to snapshot the current canvas layout. Right-click → Import Blueprint to load any saved layout.

📤 Export to vault

Export a sub-canvas as a vault HTML page via the sub-canvas context menu. Indexed and searchable immediately.

🧠 Ephemeral memory

Each canvas-built agent can have its memory toggled to ephemeral mode — no vault writes, no cross-session persistence. Good for scratch agents.

Browser & Navigation

Your vault and the web, in one place.

Two tab kinds coexist in the same tab bar. Vault tabs load local HTML via asset:// URL. Web tabs open a native child WebView on top of the app — real OS browser engine, no iframe limits.

📄 asset:// URLs

Vault pages load via Tauri's asset://localhost/… protocol. Same-origin, instant, works offline. Converted by vaultPageAssetUrl().

🔗 BacklinkPanel

Click the ⟳ icon in the address bar → right panel shows every page that links here + semantically similar pages.

🏠 Vault Home

The default tab on launch. Full-text search over the entire vault. Keyword + semantic. Category filter sidebar. Click a result to open in a vault tab.

⏱ Browser history

All visited vault URLs stored in localStorage. Click the address bar → previous URLs appear as autocomplete suggestions (<datalist>).

🌐 Native WebView

Tauri WebviewBuilder creates a real OS-level browser pane positioned over the app. Bypasses X-Frame-Options, CSP, all iframe restrictions.

🔄 URL sync

The address bar tracks the WebView's current URL via web://nav-changed events from Rust. Works for link clicks, redirects, form submissions.

🔍 SearXNG search

Type a plain query (no protocol) in any address bar → opens search results via your configured SearXNG instance. Privacy-preserving.

🔖 Bookmarks

Save any URL with the bookmark button. Bookmarks are accessible from any address bar. Stored in Tauri plugin-store.

🗺 Force-directed graph

D3 force simulation renders all vault pages as nodes and their links as edges. Color-coded by category. Drag nodes to reposition.

🖱 Click to navigate

Click any node → opens that vault page in a vault tab. Hover shows the page title and category.

🔎 Zoom & pan

Scroll to zoom, drag to pan on the graph SVG. Nodes show 20-character truncated title labels.

⚡ Auto-refresh

Listens to vault:reindexed — whenever you add a page or run the indexer, the graph updates automatically.

IDE Tools

Every tool a developer needs, locally.

No plugins. No marketplace. Shipped together and tested together.

✏️ CodeMirror 6

Syntax highlighting for 50+ languages, configurable theme, line numbers, bracket matching, auto-close.

⌨️ Vim mode

Full Vim key bindings via @codemirror/vim. Toggle in Settings.

🤖 Inline AI autocomplete

Ghost-text completions powered by the active local model. Tab to accept, Esc to dismiss.

[[ Wiki-link complete

Type [[ to get autocomplete suggestions from all vault page titles. Inserts the correct [[slug]] link.

🧜 Mermaid preview

Open any .md or .mmd file → click "Mermaid ◂" button → split-view live preview. Diagram renders offline (bundled Mermaid).

💻 Real PTY

xterm.js frontend backed by portable-pty Rust crate. Not a simulated shell — interactive programs (vim, htop, ssh) work correctly.

⬡ System Canvas

On first launch, a "Çalışan Terminaller" sub-canvas spawns with an API Server terminal (python api/server.py) and an Atlas CLI terminal pre-configured.

📁 Custom cwd

Each terminal panel can have its own working directory (meta.cwd) and an init command (meta.initCmd) that runs on spawn.

🔌 OSC handlers

Terminal supports OSC escape sequences. Used for hyperlinks and custom protocol integrations.

🎙️ Web Speech API

Uses the browser's built-in SpeechRecognition / webkitSpeechRecognition — available in Tauri's Chromium WebView. No API key, no server.

⚡ Auto-submit

After recording stops, the transcript fills the chat input as /voice [transcript] and submits automatically. No click required.

📝 Vault page creation

The /voice slash command switches to Atlas-Maker and prompts it to search for related pages, then write a new vault HTML page from your transcript.

🔴 Recording state

Microphone button turns red while recording. Interim results are available. Stop recording → transcribing → final transcript captured.

🔮 Overlay mode

Toggle with Ctrl+Alt+F. Window becomes transparent and always-on-top. Only the 148px bottom bar is opaque. Desktop is visible through the app.

🖱 Click-through

Toggle with Ctrl+Alt+P. The transparent area passes mouse clicks to the desktop (Win32 SetWindowRgn). Bar + chat balloon remain clickable.

💬 Auto chat balloon

When an agent starts "thinking" in focused mode, the chat balloon opens automatically so you see the response without switching windows.

📐 Auto-resize

Entering focused mode auto-resizes the window to full-width × 180px at the bottom of the screen. Exiting restores previous size.

Slash Commands

Type / to skip the round-trip.

Slash commands intercept the chat input before sending to the AI. They run immediately — no LLM turn needed for the common operations.

/search {query}
Runs vault_search immediately and returns results as a formatted table — no AI turn. Shows title, category, score.
Example: /search tauri webview → instant list of matching vault pages
/voice {transcript}
Auto-filled by the voice recorder. Switches to Atlas-Maker agent, then sends a prompt to check for related pages and write a new vault HTML page from the transcript.
Example: speak "meeting notes about the canvas refactor" → vault page created
/decision {text}
Appends the decision text to the active agent's office log.md. Persists across sessions for recall.
Example: /decision use onPointerDownCapture for z-index fix
/meeting {topic}
Creates a timestamped meeting note in the active agent's office folder. The agent writes the note to vault/agents/{slug}/meetings/.
Example: /meeting canvas ux review → structured meeting note saved

Quick start

Running in four commands.

Prerequisites: Node.js ≥ 18, Rust toolchain (rustup), Python 3. An LM Studio or Ollama instance is optional but recommended for AI features.

01

Clone & install — install frontend dependencies

git clone … && cd ide && npm install
02

Index the vault — creates .index/pages.json and pages.js

python tools/indexer.py
03

Generate embeddings — optional; enables semantic search and similar notes

python tools/embedder.py
04

Launch the IDE — opens Vault Home on startup

atlas-ide.bat

Development commands

  • cd ide && npm run tauri dev — start dev server
  • cd ide && npx tsc --noEmit — TypeScript check (0 errors expected)
  • cd ide/src-tauri && cargo build — verify Rust compiles
  • python tools/indexer.py — re-index after vault changes
  • python tools/embedder.py --check — verify Ollama connection
  • python api/server.py — start REST API on port 4242

Connect a local AI model

  • Start LM Studio → load a model → enable local server
  • Or: ollama run <model-name>
  • In Atlas OS → Settings → Models → set provider URL
  • Set "Chat model identifier" to the exact model name
  • For semantic search: ollama pull all-minilm
  • Test: click New Chat → ask anything
Philosophy

Four rules we don't break.

These aren't aspirations — they're constraints built into every layer.

Local first

No telemetry. No cloud sync. No account required. Vault pages live on your file system and nowhere else. The app works fully offline.

Zero lock-in

Vault pages are plain HTML readable in any browser. If Atlas OS disappears tomorrow, your data is fine. No proprietary format.

Fewer lines

No abstraction until a third use case exists. The Python side uses stdlib only — zero pip installs. Readable over clever.

Working > perfect

Ship the smallest change that delivers value. Refactor when the need is concrete. No speculative generality.
