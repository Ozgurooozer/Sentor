vault-rewrite

2026-05-19 · phase: build · owner: vault agent

Goal

Sentor vault'u, agent ofislerini birinci sınıf vatandaş olarak destekleyecek, type/scope ayrımı olan, otomatik indexlenen bir bilgi katmanına dönüştürmek.

Why now

Mevcut indexer 3-parça sabit derinlikte, vault search agent state'ini göremiyor, prototipler search'i kirletiyor. Bu altyapı düzeltilmeden agent self-RAG çalışmıyor.

Phases

  1. Faz 0 — Temizlik (Sentor sil, prototypes dışarı, Interaction Log sil)
  2. Faz 1 — Indexer/Embedder/API revize (type/scope, esnek derinlik, Ollama-only, agent snapshot endpoint)
  3. Faz 2 — Templates + sync_profiles + render_office + 3 agent ofis seed
  4. Faz 3 — Tauri vault komutları + write-guard + K16 otomatik decisions
  5. Faz 4 — Watcher + otomatik re-index
  6. Faz 5 — Frontend ofis tab + AgentSwitcher
  7. Faz 6 — vault_self_context(depth) + agent loop inject
  8. Faz 7 — /meeting slash komut + decision flow

Success Criteria

  • "Coder ne yapıyor?" → ofis kartı <150 ms.
  • "Geçen hafta ne karar verdik?" → semantic search scope agent:vault → decisions.md doğru kayıt.
  • Toplantı sonrası /meeting → 6 sn'de search'te.
  • Agent her loop başında state.md + log son 5 + aktif proje özetiyle başlar.
  • Secret içeren write → write-guard reddeder, [blocked] log girdisi.
