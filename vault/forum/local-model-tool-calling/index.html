Local Model Tool Calling Kılavuzu

dev 28 Mayıs 2026 LM Studio · Ollama · 8GB VRAM

Atlas OS canvas agent'larının araçları (tool) kullanabilmesi için hem doğru model seçimi hem de doğru altyapı gerekiyor. Bu sayfa iki altyapı yolunu, model seçim kriterlerini ve 8GB VRAM için pratik önerileri belgeliyor.

İki Farklı Tool Calling Altyapısı

Atlas OS'ta birbirinden farklı iki tool calling mekanizması var:

Chat Node (AI SDK)Orkestra Bar
Yöntem AI SDK structured function calling JSON-in-text tarama (extractCalls)
Güvenilirlik Yüksek — model API'sini kullanıyor Model bağımlı, kırılgan
Gereksinim Model function calling desteklemeli Model {"tool":"add",...} formatında JSON üretmeli

Önemli: Orkestra bar doğrudan Ollama/LM Studio'ya fetch atıyor — AI SDK kullanmıyor. Bu nedenle küçük modeller JSON formatına uymadığında araçlar sessizce çalışmıyor. Chat node ise AI SDK üzerinden structured tool calling yapıyor, bu daha güvenilir.

Pratik Test Yöntemi

Model ayarlandıktan sonra tool calling'in çalışıp çalışmadığını anlamak için:

  1. Canvas'a bir Chat node ekle
  2. Şunu yaz: "canvas_read_state tool'unu çağır ve kaç panel var söyle"
  3. Eğer canvas_read_state tool call UI'da görünüyorsa → altyapı çalışıyor
  4. Sadece düz metin yanıt geliyorsa → model tool calling'i desteklemiyor
Tarayıcı konsolu: F12 → Console'da [CANVAS:TOOL] veya [CANVAS:AI] prefix'li hataları ara. Orkestra için sessiz başarısızlıklar buraya düşüyor.

LM Studio Ayarları

Atlas IDE → Settings → Models'de:

  • LM Studio Base URL: http://localhost:1234
  • Chat Model ID: LM Studio'nun üstte gösterdiği model adını tam olarak yaz (örn: qwen2.5-7b-instruct)

Dev modunda CORS proxy otomatik devreye giriyor (vite.config.ts → /lmstudio-proxy → :1234). Production build'de LM Studio'nun kendi CORS ayarlarını açmak gerekiyor.
