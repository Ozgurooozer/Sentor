Arka Plan

C:\Atlas Os v2\JAtlas dizininde bir Electron + React prototipi incelendi. Çalışmıyor — tamamen mock. Ama görsel atmosfer mevcut Atlas OS IDE'den çok daha iyi. Karar: Atlas OS'un tüm işlevselliğini koruyarak JAtlas'ın görsel dilini sıfırdan yazacağız.

Karşılaştırma

JAtlas'ın Üstünlükleri

  • 3D canvas — z + rotX/Y/Z per panel, orbit kamera
  • Perspektif grid arka planı (rotateX 60deg)
  • Animated chat background (RAF loop, radial gradient)
  • Glassmorphic chrome (backdrop-filter: blur)
  • Temiz bottom input bar — şeffaf, kenarsız
  • Tab detach → ayrı pencere
  • Cyan accent (#4fc3f7) — daha fütüristik
  • Panel footer (renkli nokta + tip etiketi)

  • Gerçek araçlar (vault, shell, terminal, editor)

Yeni UI Mimarisi — v3

Tauri v2 + React + TypeScript korunacak. Electron'a geçiş yok. Mevcut tüm araçlar ve agent sistemi taşınacak.

1. Layout — Bottom Chrome

Mevcut top bar + sol sidebar ağır layout kalkıyor. Yeni layout:

  • Ana alan: Tüm ekran — 3D canvas veya chat view burada
  • Tab bar (32px, alt): glassmorphic backdrop-filter:blur(10px), sekme listesi + yeni sekme düğmeleri
  • Input bar (48px, en alt): şeffaf input, sol taraf tool butonları, sağ taraf gönder + çıkış
  • Üst bar: sadece canvas modunda, minimal — logo + provider pill + settings

2. 3D Canvas Sistemi

CanvasPanelNode'a yeni alanlar:

  • z: number — derinlik pozisyonu
  • rotX: number, rotY: number, rotZ: number — CSS 3D döndürme (derece)

Canvas scene:

  • perspective: 1200px + transform-style: preserve-3d container
  • Sahne rotasyonu: sceneRot.x / sceneRot.y — arka plan sürükleyerek orbit kamera
  • Perspektif grid: rotateX(60deg) scale(3) düzlemsel grid katmanı
  • Panel transform: translate3d(x, y, z) rotateX() rotateY() rotateZ()
  • Rotation handle: her panelde hover'da görünen Z ekseni döndürme tutacağı

3. Chat Panel — Animated Background

ChatPanel'in arkasına <canvas> katmanı:

  • RAF loop: requestAnimationFrame ile sürekli güncelleme
  • Radial gradient arka plan, renk HSL zamanla yavaşça kayıyor
  • 3 adet yüzen ışık topu (createRadialGradient, rgba alpha düşük)
  • Canvas component unmount'ta anim = false ile loop durdurulacak
  • Performance: will-change: transform + sadece aktif tab'da çalışacak

4. Glassmorphic Chrome

Design system güncellenir — yeni token eklenir:

  • --glass-bg: rgba(10,10,16,0.55)
  • --glass-blur: blur(16px)
  • --glass-border: rgba(255,255,255,0.06)

Tab bar ve input bar bu token'ları kullanır. Mevcut panel ve sidebar'lar değişmez.

5. Tab Detach — Tauri Multi-Window

Tauri v2'nin WebviewWindow API'siyle her chat/canvas sekmesi ayrı native pencereye alınabilir. Electron'daki electronAPI.detachTab() yerine @tauri-apps/api/window kullanılacak. State: ayrılan sekmenin mesaj geçmişi yeni pencereye serialized olarak geçirilir.

6. Renk Seçeneği

Settings → Appearance'a "Accent" seçeneği eklenir:

  • Blue #5b8def — mevcut Atlas (default)
  • Cyan #4fc3f7 — JAtlas tarzı fütüristik

Token: --accent runtime'da swap edilir, tüm bileşenler otomatik güncellenir.

C coder 27 Mayıs 2026

V3 Dual-Window sistemi çalışır hale getirildi. Süreç boyunca birkaç önemli engel yaşandı — bunları ve mevcut mimariyi belgeliyorum.

Zorlandığımız Noktalar

1. Window tam ekran açılıyordu (race condition)
main.tsx'in setTimeout(show, 50)'si Rust'ın pencere boyutlandırmasından önce tetikleniyordu. Pencere 52px değil tam ekran açılıyordu. CSS ile konumlandırma denendi — kullanıcı boş alanı kabul etmedi. Doğru çözüm: tüm pencere kurulumunu Rust setup hook'una taşımak. React yüklendiğinde pencereler zaten doğru boyutta hazır olmuş oluyor.

2. Cross-window JS API güvenilmez
WebviewWindow.getByLabel("v3-output").show() React tarafından çağrıldığında çalışmıyordu. Tauri v2'de farklı WebviewWindow'lardan JS ile başka bir pencereyi kontrol etmek güvenilmez. Çözüm: Rust'ta v3_show_output komutu yazıldı, React invoke() ile çağırdı.

3. data-tauri-drag-region decoration'sız pencerede çalışmıyor
Windows'ta decorations: false olan pencerelerde data-tauri-drag-region HTML attribute'u güvenilir sonuç vermedi. Birkaç farklı DOM yapısı denendi. Gerçek çözüm: getCurrentWindow().startDragging() — OS'e doğrudan drag moduna gir komutu.

4. e.preventDefault() drag'i engelliyordu
startDragging() çağrısının içine e.preventDefault() koymuştuk. Bu, Tauri'nin mousedown olayını yakalamasını engelliyordu. Kaldırınca drag çalıştı.

5. Capabilities dosyası güncellenmemişti — en kritik hata
Yeni v3-input ve v3-output pencereleri src-tauri/capabilities/default.json ve desktop.json dosyalarında yoktu. Tauri'nin IPC güvenlik katmanı bu pencerelerden gelen tüm çağrıları (start_dragging, minimize, close, event.listen) reddediyordu. Hata loglarında açıkça görünmesine rağmen bu en son fark edilen sorundu. Yeni Tauri penceresi eklerken capabilities güncellenmesi zorunlu.

6. npm yerine pnpm kullanılmalı
Three.js kurulumunda npm install çalıştırıldı — proje pnpm kullandığı için hata verdi. pnpm add three @types/three ile düzeltildi.

Çalışma Şeklimizde Yanlış Olan

  • Yeni pencere oluştururken capabilities'i güncellemeyi unuttuk — bu her seferinde yapılması gereken bir adım.
  • data-tauri-drag-region sorunlu olduğu bilinen bir pattern'dı, önce startDragging() denenmeliydi.
  • Hata logları daha erken okunmalıydı — capability hatası ilk andan itibaren konsolda vardı.
  • Paket yöneticisi kontrolü yapılmadan npm install çalıştırıldı.

Frontend (App.tsx): window.location.hash'e göre hangi shell'in render edileceğine karar verir.

  • 680×52px glassmorphic bar (rgba(10,10,16,0.92) + blur(28px))
  • Sol tutacak: onMouseDown → startDragging()

C coder — 10-uzman araştırma toplantısı özeti 27 Mayıs 2026

4 araştırma ajanı paralel çalıştırıldı: Three.js uzmanı, Tauri mimari uzmanı, glassmorphism CSS uzmanı, görsel yön uzmanı. Bulgular uygulandı. Aşağıda sonuçlar ve alınan kararlar.

1. Cam Efekti — Kök Sorun ve Çözüm

Sorun: body { @apply bg-background } tamamen opak bir arka plan uyguluyor. Tauri penceresi transparent: true olsa da CSS bunu örtüyordu. backdrop-filter masaüstüne değil, opak body'ye blur uyguladığı için cam efekti görünmüyordu.

Çözüm (uygulandı):

  • V3 shell bileşenleri document.documentElement.setAttribute("data-v3","true") çağırır
  • globals.css: html[data-v3="true"], body { background: transparent !important }
  • Rust lib.rs: window-vibrancy crate ile apply_mica(win, Some(true)) — Windows 11 OS compositor düzeyinde Mica efekti
  • Sonuç: masaüstü içeriği pencerenin arkasından gerçekten görünür; CSS backdrop-filter Mica yüzeyine ek katman ekler

2. Three.js Kritik Düzeltme — premultipliedAlpha

Tauri şeffaf pencerelerinde premultipliedAlpha: true (Three.js default) yarı-saydam parçacıklarda masaüstü arka planına karşı koyu hale yol açıyor. Çözüm: new THREE.WebGLRenderer({ ..., premultipliedAlpha: false }). Bu değişiklik V3OutputShell.tsx'e uygulandı.

3. Parçacık Davranış Sistemi (3 durum)

Parçacıklar artık üç farklı duruma sahip:

  • Idle drift: Perlin benzeri yavaş sürüklenme, 0.972 drag katsayısı (değişmedi)
  • Query burst: Kullanıcı mesaj gönderdiğinde tüm parçacıklar merkezden dışa doğru patlama hareketi yapar (queryBurstRef artar, animate loop yakalar)
  • Thinking pulse: AI düşünürken her ~2.4s'de parçacıklar merkeze doğru çekilir (dalga); drag katsayısı 0.980'e yükselir (parçacıklar daha canlı kalır)

4. Atlas Logo Thinking Animasyonu

AI aktif iken logo etrafında conic-gradient arc döner (1200ms linear). Teknik: 90°'lik mavi arc, radial-gradient mask ile halka şeklinde kesilir. @keyframes atlas-thinking CSS animasyonu globals.css'e eklendi. Logo kendisi dönmez — sadece arc döner.

5. Pencereler Arası Mimari — emitTo

Araştırma bulgusu: emit() tüm pencerelere broadcast yapar; emitTo("v3-output", ...) ise aynı süreç içinde Rust'tan geçmeden doğrudan hedef pencereye iletir. Değişiklik uygulandı: V3InputShell artık emitTo("v3-output", "v3:user-message", payload) kullanıyor. Önerilen namespace standardı:

  • v3:user-message — input → output (kullanıcı mesajı)
  • v3:chunk — output → araç panelleri (AI token akışı)
  • v3:wire-data — canvas → output (canvas wire bağlantısı)
  • v3:window-ready — broadcast (lifecycle)

6. Canvas → V3 Pencere Bağlantısı (Gelecek)

CanvasPanelNode'a windowLabel?: string alanı eklenecek. Bir canvas paneline wire bağlandığında, canvasStore emitTo(panel.meta.windowLabel, "v3:wire-data", data) çağıracak. Rust komutu gerekmez — aynı origin, emitTo JS'ten direkt çalışır. Bu ilerleyen fazlara ertelendi.
