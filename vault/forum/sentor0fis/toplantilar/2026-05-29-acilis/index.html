Gündem

Sentor0fis ofisini Sentor vault'una nasıl kurarız ve ilk 3 öncelik ne olmalı? Kısıt: zero-dep, dark theme, token-güvenli, MVP-first.

Özet (≤250 kelime)

Çelik ofisin tek kaynaktan render edilmesini şart koştu: config verisi, HTML sadece görüntüleyici — böylece persona/kural eklemek dosya düzenlemeden olur. Zeki ekstra dosya istemedi: "iki config (js+json) yeter, üçüncüsü ölü kod." Mert personalara tüm repoyu vermeyi reddetti: ≤200 token context_digest + sıkıştırılmış pozisyon yeter; toplantı maliyeti persona×tur. Sena hiçbir personaya yazma yetkisi verilmemesini istedi (şefler karar alır, kod yazmaz); sadece Moderatör panoyu günceller. Can ultracode modunda kararların bir kritik subagent ile yırtılmasını (self-check) önerdi. Deva ofis sayfasının Sentor dark/border-only sistemine birebir uymasını vurguladı. Nazım render'ın O(n) ve istemci-tarafı olmasını yeterli buldu. Ege kurulum sonrası tek komutla python tools/indexer.py ile aranabilir olmasını istedi. Yıldız digest'in ne zaman tazeleneceğinin ölçülebilir olmasını (materyal değişiklik) istedi. Moderatör motoru sıralı (varsayılan) tutup ultracode'u opsiyonel + token uyarılı bıraktı.

Kararlar

K1 — Data-driven ofis. Ofis config.js/config.json'dan render edilir; HTML görüntüleyici. Gerekçe: persona/kural eklemek kod düzenlemeden olsun (Çelik).
K2 — Ephemeral personalar. Her persona memory:"ephemeral"; yalnız digest + pozisyon listesi alır. Gerekçe: token sınırı + sıfır geçmiş (Mert/Sena).
K3 — Sıralı varsayılan. Motor sıralı; ultracode opsiyonel ve token-uyarılı. Gerekçe: her agent çalıştırabilsin, maliyet kontrollü (Ege/Moderatör).
K4 — İlk 3 öncelik. (1) context_digest akışı, (2) token guard eşikleri, (3) canvas blueprint. Zengin görselleştirme ertelendi. Gerekçe: MVP-first; görselleştirme düşük oy (2.2).
