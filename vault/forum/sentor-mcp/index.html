MCP Entegrasyonu

Sentor 26 Mayıs 2026 Phase F · v0.6

MCP (Model Context Protocol) sayesinde Claude Code, Cursor, Cline gibi dış AI araçları Sentor vault'unu okuyabilir ve canvas'ı kontrol edebilir.

İki Yüzlü MCP

YönMekanizmaAraç

Dış → Canvas (yazma).mcp-queue.json dosya köprüsüapi/server.py + IDE

tools/mcp_server.py — Önerilen

Sıfır bağımlılık, stdio JSON-RPC. Claude Code, Cursor, Cline, Continue ile çalışır.

# .mcp.json (proje kökü)
{
  "mcpServers": {
    "sentor": {
      "command": "python",
      "args": ["C:/Sentor/tools/mcp_server.py"],
      "env": { "SENTOR_VAULT_ROOT": "C:/Sentor" }
    }
  }
}

Araçlar:

  • vault_search — keyword arama
  • vault_read — bir sayfanın tam metni
  • vault_categories — kategori listesi
  • vault_pages — tüm sayfa listesi
  • canvas_get_state — anlık canvas snapshot
  • canvas_add_node, canvas_remove_node, canvas_connect, canvas_update_node, canvas_clear — canvas kontrolü (queue)
  • canvas_screenshot — ekran görüntüsü → base64 PNG
Bilinen sorun: İki MCP sunucusu var — tools/mcp_server.py (önerilen) ve mcp/server.py (HTTP transport ekli, 747 satır). İşlevsel örtüşme var. Konsolidasyon kararı bekleniyor.

Queue Bridge — Canvas Kontrolü

Dış araçlar canvas'ı doğrudan değiştiremez (IDE Tauri process içinde çalışıyor). Bunun yerine queue dosyasına komut yazılır:

# Akış
Dış araç (Claude Code)
  → POST /api/ide/canvas/panels
  → api/server.py → .mcp-queue.json'a komut ekler
  → Rust file watcher (notify crate) → sentor:mcp-cmd event
  → IDE frontend drain → canvasStore.addPanel(...)
30 saniye fallback: File watcher olayı kaçırırsa IDE her 30 saniyede de queue'yu kontrol eder. İki katmanlı güvence.

Canvas State Export

Canvas her değiştiğinde .ide-state.json'a otomatik kaydedilir. Dış araçlar bu dosyayı okuyarak güncel canvas durumunu görebilir.

# useMcpBridge.ts tarafından tetiklenir
mcp_export_state() → .ide-state.json
mcp_dequeue()      → .mcp-queue.json'u oku + temizle
