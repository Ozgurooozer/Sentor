Sentor V3 Mimarisi — Detaylı Teknik Analiz

mimari v3-canvas 27 Mayıs 2026 Phase H

Bu konu Sentor'un V3 canvas sisteminin tam mimari analizini içerir: katman yığını, wire sistemi, Canvas AI agent tasarımı, store yapısı ve V3InputShell köprüsü. Phase I temizliğine zemin hazırlamak amacıyla yazıldı.

2. Veri Modeli

CanvasPanelNode

interface CanvasPanelNode {
  id: string          // crypto.randomUUID()
  type: PanelType     // "terminal" | "chat" | "note" | ... (23 tip)
  x, y: number       // canvas-space koordinatlar
  width, height: number
  zIndex: number
  title: string
  meta: Record<string, unknown>  // tip-spesifik veri
  pinned?: boolean    // viewport'a sabitlenmiş
  minimized?: boolean // dock'a küçültülmüş
  status?: "idle" | "running" | "error" | "done"
}

Connection (Wire)

interface Connection {
  id: string
  fromPanel: string   fromSide: PortSide   fromPort?: string
  toPanel: string     toSide: PortSide     toPort?: string
  kind?: "data" | "context" | "trigger"
  charLimit?: number  // varsayılan 4000 karakter
}

Wire türleri görsel ve semantik olarak ayrılmıştır:

  • data Mavi — açık değer aktarımı, chat badge satırında görünür
  • context Mor — sessiz oto-context, her prompt'a eklenir, badge göstermez
  • trigger Yeşil — sadece sinyal, veri taşımaz

3. State Yönetimi — canvasStore

canvasStore.ts (668 satır) Zustand ile yazılmış monolitik store. Per-vault kalıcılık için @tauri-apps/plugin-store kullanır — her vault kendi sentor-canvas-{hash}.json dosyasını alır.

// Temel state

// Multi-canvas

// Split view
isSplit: boolean

setOutputData(id, {kind, value}) — panel çıktısını meta.outputData'ya yazar. Downstream wire'lar useAllIncomingWireData(panelId) hook'u ile bunu okur, charLimit'e göre keser ve WireBlock[] döndürür.

4. Port Sistemi

portDefs.ts her panel tipinin named port'larını tanımlar:

// Örnek: terminal panelinin portları
terminal: {
  inputs:  [text("cmd", "cmd", "data"), trig("trigger", "run")],
  outputs: [text("stdout", "stdout", "data")]
}

// Her port: { id, label, kind: ConnectionKind, dataType: PortDataType }
// dataType: "text" | "image" | "json" | "trigger" | "any"

namedPortPoint(panel, side, index, total) — port dot'unun canvas-space koordinatını hesaplar (header yüksekliği çıkarıldıktan sonra eşit dağılım). portIndex() ve portKind() bağlantı doğrulaması için kullanılır.

5. Canvas AI Asistanı — orkestraStore

Canvas'ı doğal dille yöneten AI katmanı. AI SDK kullanmaz — Ollama (/api/chat stream) veya LM Studio (/v1/chat/completions) API'larına doğrudan fetch atar. Bu sayede herhangi bir OpenAI uyumlu local model çalışır.

Alias Sistemi (UUID sorunu çözümü)

Local model'lar UUID (550e8400-e29b-41d4-...) kopyalarken hata yapar. Çözüm: her send()'de ve her tool yürütmesinde canvas state'i kısa alias'lara eşlenir:

// buildSystem() veya rebuildAliases() çağrısında:
visible.forEach((p, i) => { _aliasToId[`n${i+1}`] = p.id });
// → n1, n2, n3 … → gerçek UUID'ler

// execTool başında her seferinde yeniden inşa:
function execTool(call: ToolCall): string {
  rebuildAliases(); // yeni eklenen node'lar da n3, n4 olur
  const rid = (v) => resolveId(String(v ?? ""));
  ...
}

resolveId() bulunamayan alias için "" döndürür ("n1" string'i değil). Bu sayede run tool'u geçersiz id aldığında canvas'taki ilk terminale otomatik fallback yapar.

JSON Extraction

Model yanıtı düz metin + gömülü JSON karışımıdır. Brace-balanced scanner tüm {"tool":...} nesnelerini bulur — multi-line JSON'ı da doğru parse eder. Regex yerine scanner kullanılmasının nedeni: yerel model'lar arasında indentation tutarsızlığı.

6. V3InputShell → Canvas Köprüsü

V3InputShell ayrı bir Tauri penceresidir (always_on_top, şeffaf arkaplan). Canvas'a bağlı değilken normal AI assistant gibi çalışır. ⊞ butonu ile canvas'a "link" edilebilir:

V3InputShell (v3-input window)
  → emit("sentor:canvas-prompt", { text })
    → CanvasAppShell listens via Tauri listen()
      → useOrkestraStore.send(text, model, ...)
        → orkestraStore tool execution
          → window.dispatchEvent("sentor:terminal-trigger")
            → V3TerminalPanel PTY write

Bağlantı kesilebilir: sentor:canvas-unlink event'i v3InputActive flag'ini sıfırlar. Konuşma geçmişi V3InputShell penceresinde toggle edilebilir bir panel olarak gösterilir — pencere otomatik boyut değiştirir (Tauri setSize + setPosition).

7. Terminal → PTY Köprüsü

Canvas terminal paneli ile PTY arasındaki veri akışı:

sentor:terminal-trigger event
  { panelId: UUID, text: string }
    → V3TerminalPanel handler
      → paneRef.current?.write(text + "\r")
        → TerminalPaneHandle.write()
          → useTerminalSession.write()
            → sessions.get(leafId)?.pty?.write(data)
              → Tauri PTY bridge (pty-bridge.ts)
                → Rust pty module (shell_write)

leafId panel UUID'sinden türetilmiş stabil bir sayıdır (getLeafId()). PTY başlangıçta lazy init edilir — terminal görünür olduğunda ilk fit çağrısından sonra açılır. Eğer PTY henüz bağlı değilse write sessizce undefined döndürür.

8. Wire Veri Akışı

Producer panel
  → meta.outputData = { kind: "text"|"image"|"json", value }
  → setOutputData(id, data)          // canvasStore action

Consumer panel (örn. chat)
  → useAllIncomingWireData(panelId)  // tüm upstream bağlantılar
    → connections.filter(c => c.toPanel === panelId)
    → her bağlantı için: source.meta.outputData || source.meta.snapshotData
    → charLimit'e göre kes (varsayılan 4000)
    → WireBlock[] döndür: { panelId, panelType, connectionKind, data, toPort }

Chat panel
  → toPort === "context" → her prompt'a eklenir (sessiz)
  → toPort === "data"    → badge satırında gösterilir
  → toPort === "trigger" → veri taşımaz, sadece çalıştır

9. Render Pipeline

Canvas'ta her frame'de neler render edilir:

  • V3CanvasBg: Three.js WebGL canvas (pointerEvents: none — kritik, aksi halde xterm'i engeller). Perspektif grid + parçacıklar + UnrealBloom. ResizeObserver ile composer yeniden boyutlandırılır.
  • Transform layer: translate(vp.x, vp.y) scale(vp.scale) div'i. Tüm V3CanvasNode'lar bu katmanda — tek CSS transform ile pan/zoom.
  • V3WireLayer: SVG, transform layer içinde. Bezier path'ler resolveEndpoint() ile port noktalarından hesaplanır. Pending connection için anlık cursor tracking.
  • PinnedPanelsPortal: React portal — pinned node'ları transform layer dışında, position: fixed ile render eder.
