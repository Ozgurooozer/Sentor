Write-Guard Pattern: AI-İnsan Ortak Yazarlığı

Kod Kalitesi guvenlik · 22 Mayıs 2026 1 yanıt

Mevcut Durum

Vault, secret tespiti için regex deny-list kullanıyor:

  • sk-... — OpenAI API anahtarları
  • ghp_..., github_pat_... — GitHub token'ları
  • xoxb-... — Slack token'ları
  • -----BEGIN ...----- — PEM blokları

Ayrıca <!-- agent:start --> / <!-- agent:end --> blok protokolü ile AI agent'ların sadece belirlenen bölümleri değiştirmesi sağlanıyor. Agent blok dışına dokunamıyor.

Tartışma

  • Regex deny-list yeterli mi, yoksa allow-list daha güvenli olur mu?
  • Blok protokolü hangi edge case'lerde kırılır? (iç içe bloklar, agent crash, yarım yazma)
  • Atomic write (write-tmp + rename) burada koruma sağlıyor mu?

Regex deny-list yeterli mi? Bence allow-list daha güvenli olur. Bir API anahtarı yeni bir formatta gelirse deny-list'ten kaçabilir. Ama allow-list de bakım maliyeti getirir. İkisinin kombinasyonu en iyisi olabilir: bilinen tehlikeli pattern'leri blockla, bilinen güvenli pattern'lere izin ver, gerisini logla.
