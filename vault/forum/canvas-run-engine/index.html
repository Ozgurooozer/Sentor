Canvas Run Engine + Gate Panel

Sentor Phase M 28 Mayıs 2026

Canvas'taki node'ları wire bağlantılarını takip ederek sırayla çalıştıran topological execution engine ve "sinyal kapısı" görevi gören yeni Gate panel tipi. cli-final prototipindeki Circuit/DecisionGate fikirlerinden ilham alındı.

Motivasyon: cli-final Prototipi

prototypes/cli-final/ klasöründe daha önce yazılmış bir Node.js CLI prototipi var. İki önemli fikir içeriyordu:

  • DecisionGate: Sinyali alan ve bir AI kriterine göre "geçmesine izin ver / kes" mantığı. Mevcut if/else panelinden farklı — if/else sadece yönlendiriyor, gate ise downstream'i tamamen durdurabiliyor.
  • canvas.run(entryNodeId): Giriş node'undan başlayıp wire'ları takip ederek sırayla çalıştırma. Canvas'ta ▶ Run butonu vardı ama gerçek bir execution engine yoktu.
Prototipin durumu: Doğrudan import edilemez (Node.js, IDE ise Tauri/React). Ama DecisionGate ve topological run mantığı sentor'a uyarlandı.

Canvas Run Engine

ide/src/modules/canvas/canvasEngine.ts

Çalışma mantığı

  1. Pinned/minimized olmayan paneller alınır
  2. Data/context wire'ları üzerinden Kahn's BFS topo-sort yapılır (trigger wire'lar sıralamayı etkilemez)
  3. Paneller bağımlılık sırasına göre çalıştırılır
  4. Her panel çıktısını setOutputData ile store'a yazar → downstream wire'lar güncel veriyi görür
  5. Gate kapanırsa tüm downstream paneller bloke edilir
  6. 3 saniye sonra tüm status indikatörleri idle'a döner

Panel execution mantığı

▶ Run butonu

V3CanvasTopBar'daki Run butonu artık window.dispatchEvent(canvas:run) yerine doğrudan runCanvas() çağırıyor. Çalışırken loading spinner gösteriyor, tamamlanınca normale dönüyor.

Gate Panel

ide/src/modules/canvas/GatePanel.tsx

gate — Circuit gate · sinyal geçir veya bloke et

Portlar

YönPortAçıklama
GirişsignalDeğerlendirilecek veri sinyali (any)
GirişconditionOpsiyonel: wire'dan gelen koşul (meta.criteria'yı override eder)
ÇıkışpassGate açıksa sinyal buradan geçer
ÇıkışrejectedGate kapalıysa sinyal buradan çıkar

Değerlendirme modları

ModKoşul
truthySinyal boş/false/0/null değilse geç (varsayılan)
not-emptySinyal en az 1 karakter içeriyorsa geç
containsSinyal criteria metnini içeriyorsa geç
regexSinyal regex pattern'ına uymaktaysa geç

if-else ile farkı

if-else: Veriyi iki yönden birine yönlendirir. Her iki branch da downstream çalışır, sadece farklı veri alır.

gate: pass portunun downstream'ini ya tamamen çalıştırır ya da tamamen bloke eder. Run engine, bloke edilen panelleri sıra gelince atlıyor.

Örnek kullanım:
Input → Gate (contains: "hata") → Terminal
Input'tan gelen metin "hata" içermiyorsa Gate kapanır, Terminal hiç çalışmaz.

Durum göstergesi

  • 🟢 yeşil + "geçti ▸" — son run'da gate açıktı
  • 🟠 turuncu + "bloke ✕" — son run'da gate kapalıydı, downstream atlandı
  • ⚪ gri + "bekliyor" — henüz run edilmedi
