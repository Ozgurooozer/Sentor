1. Mevcut Canvas Durumu (Kod Analizi)

Canvas Analisti

Sentor'un mevcut canvas'ı temelden sağlam. Çalışan pattern'lar korunacak; bazı kritik eksikler kapatılacak.

Çalışan ve Korunacaklar

  • CSS transform container — tek div'e translate(x,y) scale(s) uygulamak. Figma, tldraw, Excalidraw ile aynı yaklaşım.
  • Wheel handler'da getState() — React render döngüsünü bypass ederek doğrudan store'dan okuma. Zoom için doğru pattern.
  • Pointer capture drag — setPointerCapture ile sürükleme; panel sınırlarında pointer kaybı yok.
  • SVG overlay bağlantı çizgileri — canvas koordinat sisteminde cubic bezier. SVG → React reconciler entegrasyonu doğru.
  • schedulePersist debounce (200ms) — drag sırasında disk yazma fırtınası önlüyor.
  • Per-vault hash persistence, multi-canvas dosya izolasyonu.
  • loadBlueprint ID remapping — import/blueprint için sağlam.

Performans Sorunları

  • panels.map() her güncelleme — panel başlığı edit, drag, terminal output her birinde tüm panels dizisi map'leniyor. 50+ panelde yavaşlar.
  • connections.filter() her render — her CanvasPanel kendi bağlantılarını bulmak için tüm connections'ı filter'lıyor. O(n·m) karmaşıklık.
  • ConnectionLayer'da new Map(panels.map(...)) — her render'da yeniden oluşturuluyor. useMemo eksik.
  • Viewport culling yok — ekran dışındaki tüm panel'ler render ediliyor.
  • bringToFront her tıklamada — her panel click'te store yazma + full re-render tetikliyor.

Kritik Eksikler

  • Seçim sistemi yok — selectedPanelIds store'da mevcut değil. Delete key, multi-select, marquee imkânsız.
  • Undo/redo yok — canvas mutasyonlarında geçmiş stack tutulmuyor.
  • Dot grid pan/zoom ile hareket etmiyor — backgroundSize ve backgroundPosition statik; grid koordinat sisteminden kopuk.
  • stickman type gap — PANEL_DEFAULTS'ta var, types.ts'te yok.
  • Terminal setInterval — her 3 saniyede değişmese de store'a yazıyor.

2. Infinite Canvas Mimarisi (Online Araştırma)

Web Araştırmacı

Kanıtlanmış Koordinat Sistemi

Figma, tldraw, Excalidraw, React Flow — hepsi aynı "camera" modelini kullanıyor:

// Sentor zaten bunu uyguluyor — format doğru
interface Camera { x: number; y: number; z: number; } // z = zoom

screenToCanvas: (pt, cam) => ({ x: pt.x / cam.z - cam.x, y: pt.y / cam.z - cam.y })
canvasToScreen: (pt, cam) => ({ x: (pt.x + cam.x) * cam.z, y: (pt.y + cam.y) * cam.z })

// CSS transform — scale ÖNCE translate (sıra önemli):
transform: `scale(${cam.z}) translate(${cam.x}px, ${cam.y}px)`
Sentor mevcut canvas doğru koordinat sistemini kullanıyor. Tamamen yeniden yazılmasına gerek yok — eksikler kapatılacak.

Library vs Sıfırdan Yazma

Karar

Mevcut hand-rolled canvas korunacak. React Flow / XY Flow'a geçilmeyecek. Sebebi: Sentor node'ları derin Tauri entegrasyonu gerektiriyor (PTY terminal, native WebView, wire data forwarding). React Flow bu entegrasyonları kırar. Mevcut canvas zaten doğru temeller üzerinde.

SVG vs Canvas2D Wire Rendering

SVG overlay (mevcut yaklaşım) doğru seçim. Canvas2D için her frame'de clearRect + full redraw gerekir. SVG, React reconciler ile entegre çalışır — sadece değişen wire güncellenir. 50 panel / 100 wire için yeterince hızlı.

Bezier Wire Formülü

function edgePath(x1, y1, x2, y2) {
  const dx = Math.abs(x2 - x1) * 0.5;
  const c1x = x1 + dx, c1y = y1;   // output kontrol noktası
  const c2x = x2 - dx, c2y = y2;   // input kontrol noktası
  return `M ${x1} ${y1} C ${c1x} ${c1y}, ${c2x} ${c2y}, ${x2} ${y2}`;
}
// Geriye giden bağlantılar için dx'i minimum 80px'e sabitle

Sentor mevcut implementasyon: dist * 0.45 — bu iyi ayarlanmış, değiştirilmeyecek.

8. Viewport Culling Mimarisi

50+ node için. tldraw'ın yaklaşımı: DOM ref ile display:none — React reconciler'ı bypass eder, state değiştirmez.

// InfiniteCanvas.tsx — her camera move'da
function isVisible(panel: CanvasPanelNode, vp: Viewport, vw: number, vh: number) {
  const sx = panel.x * vp.scale + vp.x;
  const sy = panel.y * vp.scale + vp.y;
  const buffer = 200;
  return sx < vw + buffer && sx + panel.width * vp.scale > -buffer
      && sy < vh + buffer && sy + panel.height * vp.scale > -buffer;
}

// CanvasPanel ref'e doğrudan:
panelRef.current.style.display = isVisible(panel, vp, W, H) ? "" : "none";

9. Undo/Redo Mimarisi (Sonraki Faz)

// canvasStore.ts
type Snapshot = { panels: CanvasPanelNode[]; connections: Connection[] };
undoStack: Snapshot[];  // max 50
redoStack: Snapshot[];

// Her mutasyon öncesi:
function withHistory(fn: () => void) {
  const snap = { panels: [...get().panels], connections: [...get().connections] };
  get().undoStack.push(snap);
  if (get().undoStack.length > 50) get().undoStack.shift();
  get().redoStack = [];
  fn();
}

// Undo: pop undoStack → push to redo → restore
// Redo: pop redoStack → push to undo → restore
