AI Agent Sistemi

Atlas OS 26 Mayıs 2026 Phase B · Phase G

Atlas OS'ta üç yerleşik agent vardır. Hepsi yerel AI modelleriyle çalışır (LM Studio veya Ollama). API key gerekmez, internet bağlantısı gerekmez.

Üç Yerleşik Agent

◈ Vault Agent
Araştırma ve Belgeleme
Vault sayfalarını arar, okur, aralarındaki bağlantıları keşfeder. "Atlas OS'un MCP sistemi nasıl çalışıyor?" gibi sorulara vault'tan yanıt üretir.
Araçlar: vault_search, vault_read, vault_semantic, web_search, web_fetch
⚙ Coder Agent
Kaynak Kod Düzenleme
Kaynak dosyaları okur, düzenler, terminal komutları çalıştırır. Mutating araçlar (write_file, edit, bash) kullanıcı onayı gerektirir.
Araçlar: read_file, write_file, edit, multi_edit, bash_run, grep, glob
✦ Atlas-Maker Agent
Vault Sayfası Yazma
Sıfırdan vault sayfası oluşturur. Tasarım sistemine uygun HTML üretir, onay kartı gösterir, onaylanırsa vault'a yazar ve otomatik re-index çalıştırır.
Araçlar: vault_write, vault_search, vault_read

Self-RAG — Agent Kendi Notlarını Okur

Her agent açılışta kendi "ofis" klasörünü okur:

vault/agents/vault/     ← Vault agent'ın notları
vault/agents/coder/     ← Coder agent'ın notları
vault/agents/atlas-maker/ ← Atlas-Maker'ın notları
Ne işe yarıyor: Agent ofis klasöründe önceki kararlar, tercihler, devam eden görevler bulunabilir. Yeni bir oturumda agent "geçen sefer ne konuşmuştuk?" bağlamını kendi notlarından çeker. vault_self_context Tauri komutu bunu otomatik yapar.

Subagent Sistemi

Ana agent iki tip subagent kullanabilir:

  • explore — Dosya/vault araştırma subagenti. Dosya bul, içerik oku, arama yap.
  • general — Genel amaçlı subagent. Hesaplama, analiz, yazma görevleri.
// Subagent çağırma (agent döngüsünden)
{
  type: "explore",
  task: "Find all TypeScript files that import canvasStore"
}

Provider Yapılandırması

Her agent için ayrı model seçilebilir:

ProviderURLNotlar
LM Studiolocalhost:1234/v1OpenAI uyumlu, yerel GPU
Ollamalocalhost:11434/v1Llama, Qwen, Gemma modelleri
Dev modunda CORS: Vite proxy /lmstudio-proxy ve /ollama-proxy path'lerini ilgili portlara yönlendirir. Üretim Tauri build'inde provider kendi CORS ayarından sorumludur.

Araç Onay Politikası

  • Otomatik çalışır: read_file, list_directory, grep, glob, vault_search, vault_read
  • Kullanıcı onayı gerekir: write_file, edit, multi_edit, create_directory, bash_run
  • edit/multi_edit ayrıca read-before-edit kuralını zorunlu kılar (readCache kontrolü)
