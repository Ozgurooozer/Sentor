CLI ve Pipeline

Sentor 26 Mayıs 2026 cli/ · Otomasyon

IDE açmadan vault araması, AI sohbeti, pipeline çalıştırma ve cron tetikleme. Tüm CLI araçları Python stdlib kullanır — pip, sanal ortam, derleme gerekmez.

main.py — Ana CLI

python cli/main.py <komut>

  index                      # vault'u yeniden indexle
  search "sorgu"             # keyword araması (TF-IDF benzeri puanlama)
  list [kategori]            # sayfaları listele, opsiyonel kategori filtresi
  open kategori/slug         # vault sayfasını terminalde göster
  serve [port]               # API sunucusunu başlat (varsayılan 4242)
  chat                       # Ollama ile etkileşimli AI sohbeti

chat komutu — Ollama timeout

Ollama yanıt vermezse CLI sonsuza kadar bekleyebilir. Bu yüzden HTTP düzeyinde timeout uygulanır:

# cli/main.py içinde
_client = ollama.Client(timeout=120)   # 120 saniye HTTP timeout
response = _client.chat(model=model, messages=messages, tools=tools)
Neden önemli: ollama.chat() modül fonksiyonu timeout parametresi almaz — bu davranış model parametrelerini etkiler, HTTP bağlantısını değil. ollama.Client(timeout=...) tek doğru yol.

Pipeline Sistemi — cli/pipeline.py

Pipeline'lar JSON dosyalarında tanımlanır. Her pipeline bir dizi adımdan oluşur:

# vault/pipelines/daily-summary.json
{
  "id": "daily-summary",
  "steps": [
    { "type": "task", "task_id": "collect-notes" },
    { "type": "notify", "message": "Özet hazır: {{output}}" }
  ],
  "on_error": "stop"   # "stop" | "continue" | "retry"
}
Adım tipiAçıklama
taskBir Sentor görevini çalıştırır (task_id ile)
notifySistem bildirimi gönderir (message şablonu)
shellKabuk komutu çalıştırır (command)
sys.exit kaldırıldı: Eski kod _load_pipeline()'da bulunamayan pipeline için sys.exit(1) çağırıyordu. Bu arka plan thread'ini de öldürüyordu. Artık FileNotFoundError fırlatılıyor, on_error politikası devreye giriyor.

Sentor Entegrasyonu — cli/sentor.py

Sentor, yerel Flowise örneğini (http://127.0.0.1:3000) CLI'dan yönetir:

python cli/sentor.py list              # mevcut flow'ları listele
python cli/sentor.py run <task_id>    # bir görevi çalıştır
python cli/sentor.py show <task_id>   # görev detaylarını göster

Görev bulunamadığında FileNotFoundError fırlatılır — pipeline on_error politikası bunu yakalayabilir.

serve-daemon — cli/serve_daemon.py

Pipeline'ları cron zamanlamasıyla veya dosya değişikliği tetikleyicisiyle otomatik çalıştıran arka plan servisi:

python cli/serve_daemon.py start       # daemon'ı başlat
python cli/serve_daemon.py stop        # durdur
python cli/serve_daemon.py status      # çalışan pipeline'ları göster

Eşzamanlılık koruması

Tetikleyici hızlı ateşlenirse aynı pipeline birden fazla kez başlayabilirdi. Artık modül düzeyinde kilit var:

_running_lock = threading.Lock()
_running: set = set()

def _fire(pipeline_id, ctx=None):
    with _running_lock:
        if pipeline_id in _running:
            _log(f'pipeline [{pipeline_id}] already running, skipping', 'INFO')
            return
        _running.add(pipeline_id)

    def _go():
        try:
            run_pipeline(pipeline_id, ctx)
        finally:
            with _running_lock:
                _running.discard(pipeline_id)

    threading.Thread(target=_go, daemon=True).start()
finally garantisi: Pipeline başarılı da olsa hata da verse _running setinden çıkarılır. Bir sonraki tetikleyici normale döner.

sentor.bat — Windows All-in-One

Proje kökündeki sentor.bat interaktif menü sunar:

sentor.bat

  [1] API sunucusunu başlat (port 4242)
  [2] IDE başlat (Tauri dev)
  [3] Vault'u yeniden indexle
  [4] AI chat (Ollama)
  [5] Serve daemon başlat

sentor.bat ise API sunucusunu arka planda başlatıp ardından IDE'yi açar — iki adımı tek komuta indirger.

sys.path Stratejisi

CLI dosyaları birbirini import edebilmek için sys.path'e ekleme yapar. Eski kod bunu her fonksiyon çağrısında tekrar yapıyordu. Artık modül seviyesinde, bir kez yapılır:

# cli/pipeline.py — modül seviyesi
_cli_path   = str(ROOT / 'cli')
_tools_path = str(ROOT / 'tools')
for _p in (_cli_path, _tools_path):
    if _p not in sys.path:
        sys.path.insert(0, _p)
