Proje Felsefesi

∅ Zero Dependency

Python tarafında hiçbir pip paketi kullanılmaz. stdlib yeterlidir. npm bağımlılıkları minimal tutulur.

📍 Local-First

Her şey yerel çalışır. Cloud API key'leri opsiyoneldir. Temel özellikler için internet gerekmez.

📄 HTML as Source of Truth

Vault sayfaları düz HTML dosyalarıdır. Herhangi bir tarayıcıda açılabilir. Veri kilitlenmesi yoktur.

🔧 Pragmatic Over Perfect

"Working > perfect" felsefesi. Bilinen TODO'lar KALAN_ISLER.md'de tutulur. Hatalar belgelenir ve ertelenir.

🧩 Phase-Based Development

Tüm geliştirme fazlara bölünmüştür (Phase 0-F). Her faz bağımsız teslim edilebilir.

🔐 Security by Default

Bearer token auth, CSP header, SSRF koruması, path traversal koruması, atomic writes.

  • Backlink ve semantic search için yeterli yapı

  • Kullanıcı "python api/server.py" ile çalıştırır

Neden iki MCP sunucusu var?

Neden Zustand, Redux değil?

Vault Sistemi

Bilgi Tabanının Anatomisi

Vault, Sentor'un kalbinde yer alır. Her şey HTML dosyaları olarak başlar ve indexlenerek aranabilir hale gelir.

Veri Akışı: Yaz → İndex → Embed → Ara

vault/{category}/{slug}/index.html ← kaynak gerçek
    │
    ▼
tools/indexer.py ← 529 satır, iki-pass: extract → backlink resolve
    │ • HTMLParser ile title/meta/heading/link çıkarımı
    │ • Tip tespiti: 15 pattern-match kuralı (agent-profile, meeting, note...)
    │ • SHA-1 content_hash ile değişim tespiti
    │ • Artımlı mod: --changed-files ile sadece değişenleri işle
    ▼
.index/pages.json ← schema_version:2, 3000 karakter text, backlinks[]
.index/pages.js ← browser-loadable (window.SENTOR_INDEX)
    │
    ▼
tools/embedder.py ← 261 satır, artımlı embedding
    │ • content_hash karşılaştırması ile eski embedding'leri koru
    │ • Non-embeddable tipler: agent-log, template, agent-profile-source
    │ • Ollama yoksa no-op (keyword-only mod)
    ▼
.index/embeddings.json ← 384-dim vectors (all-minilm)
    │
    ▼
api/server.py → GET /api/search (keyword: TF-IDF) + GET /api/semantic (cosine)
ide/ → hybrid search: keyword(60%) + semantic(40%) + bonus(0.1)
ui/index.html → Fuse.js fuzzy search (browser-only, no server)

Vault Klasör Yapısı

Klasörler ve Tipleri

  • home/ — note (genel bilgi sayfaları)
  • projects/ — project (proje dokümantasyonu)
  • agents/{slug}/ — agent-office (ajan ofisleri)
  • meetings/{slug}/ — meeting (toplantı notları)
  • templates/ — template (şablon, arama dışı)
  • archive/ — archive (opt-in arama)
  • ai-design/ — note (AI tasarım desenleri)
  • html/ — reference (teknik referans)
  • logs/ — sistem logları (JSON)
  • blueprints/ — canvas blueprint dosyaları

Her Sayfada Bulunanlar

  • title — <title> etiketi
  • description — <meta name="description">
  • headings[] — h1/h2/h3 başlıkları
  • links[] — yerel bağlantılar (vault ID'ye çözülür)
  • backlinks[] — bu sayfayı referans alanlar
  • text — ilk 3000 karakter gövde metni
  • content_hash — SHA-1 (değişim tespiti için)
  • type/scope — path pattern'den türetilir

Üç Arama Modu

⌨️ Keyword

TF-IDF: title ×3, heading/desc ×2, body ×1. Lokal index'te çalışır, sıfır ağ gerektirir. scoring.py ile CLI/API/IDE ortak kullanır.

🧠 Semantic

384-dim cosine similarity. Ollama all-minilm ile sorgu embed edilir. Tüm embedding'lerle karşılaştırılır. Score > 0.1 filtresi.

🔀 Hybrid (IDE)

Önce keyword, sonuç zayıfsa semantic'e geçer. 0.6 × keyword + 0.4 × semantic + 0.1 bonus. Kaynak etiketi ekler.

Bilinen Sorunlar

embedder.py — _page_text() iki kere çağrılıyor: Aynı sayfa için hash hesaplama ve embed gönderme aşamalarında iki kez aynı fonksiyon çalışır. Ara sonuç cache'lenmeli.
indexer.py — her zaman yazıyor: build_full() içerik değişmese bile pages.json + pages.js'i diske yazar. Hash karşılaştırması eklenebilir.
Özel YAML parser: _parse_yaml_lite() sadece basit key-value ve dash listelerini destekler. İç içe YAML yapıları sessizce yanlış parse edilir.

🔴 Kritik Bulgular

⚠ İki MCP Sunucusu

mcp/server.py (753 satır) ve tools/mcp_server.py (645 satır) aynı işi yapıyor. Biri silinmeli veya diğerini import etmeli.

⚠ 45 Rust unwrap()

Her biri potansiyel panic. Özellikle lock().unwrap() pattern'i yaygın. ? operator veya unwrap_or_else kullanılmalı.

⚠ 26 sys.path.insert

pyproject.toml + pip install -e . ile çözülmeli. Şu an her çalıştırmada path pollution oluyor.

⚠ sentor.py sys.exit(1)

Task bulamayınca process'i öldürüyor. Pipeline'ın on_error mantığı çalışamıyor. Exception döndürmeli.

🟡 Orta Seviye Bulgular

🔧 Hardcoded URL'ler

localhost:11434, localhost:4242 vb. 20+ yerde tekrarlanıyor. Çevre değişkeni veya config dosyasına taşınmalı.

🔧 Queue Race Condition

tools/mcp_server.py'de _enqueue lock kullanmıyor. api/server.py'de lock var, burada yok.

🔧 TOCTOU Queue

.mcp-queue.json exists check → read → write arasında IDE queue'yu boşaltırsa veri kaybolabilir.

🔧 _page_text() Çift Çağrı

embedder.py'de her sayfa için hash ve embed aşamalarında aynı fonksiyon iki kere çalışıyor.

🔧 API Server Monoliti

api/server.py 845 satır, 42 metodlu _Handler class'ı. Route'lar ayrılmalı.

🔧 Test Kapsamı Düşük

Sadece 3 test dosyası. Custom runner (pytest değil). CI-safe olan sadece 1 test. Unit test yok.

🟢 İyi Uygulamalar

✓ TypeScript Tip Güvenliği

Sadece 1 adet any tipi bulundu. Zod şemaları, generic'ler, as const satisfies pattern'i kullanılıyor.

✓ Zero Dependency Python

Python tarafında hiç pip paketi yok. stdlib http.server ile REST API, JSON-RPC, HTML parsing.

✓ Güvenlik

Bearer auth, CSP, SSRF koruması, path traversal guard, atomic writes, panic=unwind.

✓ Dokümantasyon

CLAUDE.md (377 satır), AGENTS.md, CHANGELOG.md, KALAN_ISLER.md, sentor-durum.md — hepsi güncel.

✓ Atomic File Writes

.mcp-queue.json, .index/*.json yazmaları write-tmp + rename ile atomic. Veri kaybı önlenmiş.

✓ Artımlı İndexleme

indexer.py --changed-files ile sadece değişen dosyaları işler. embedder.py content_hash ile eski embedding'leri korur.

Neden Vault Memory Döngüsü?

Context Window Sorunu

  • LLM'lerin context window'u sınırlı (8K-200K token)
  • Uzun süreli hafıza için her şeyi prompt'a sığdıramazsınız
  • Vault çözümü: bilgiyi HTML'de sakla, ihtiyaç duyunca çağır
  • Her agent konuşması vault'a yazılır → kalıcı bellek

Döngünün Avantajları

  • Bilgi kaybolmaz — vault'ta HTML olarak yaşar
  • Context window şişmez — sadece ilgili sayfalar yüklenir
  • Full-text + semantic search ile doğru sayfa bulunur
  • İnsan da AI da aynı kaynağı kullanır
