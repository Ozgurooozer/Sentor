// 06 · tipik senaryolar
Sistemin nasıl çalıştığı, örnek üzerinden.

Beş tipik akış: agent üretmek, paralel sorgu, sentor pencere bağlama, MCP önerisi, projeye çevir.

1. Sıfırdan agent üret Sentor
Kullanıcı: "vault'taki tüm meeting notlarımı özetleyen bir agent yap"
→ Orkestra Sentor'u tetikler. Sentor canvas'ı açılır, seed context yüklenir.
→ Sentor önerir: name: meeting-summarizer, tools: [vault_search, vault_read, llm], base: Vault.
→ Kullanıcı onaylar. agent_spawn çağrılır. Yeni agent agentsStore'a yazılır, sidebar'da görünür.
2. İki paneli tek agent'a bağla Canvas
Canvas'ta açık: vault/projects/canvas-plan.html + Terminal (cargo check çıktısı).
→ Sağ tık → Agent Ekle. Panel açılır.
→ "İsim: build-doctor, Görev: derleme hatalarını canvas-plan ile karşılaştır".
→ Vault node'unun out port'undan agent in port'una sürükle. Terminal node'undan da bağla.
→ Agent'a soru sor: "Bu hata plan ile uyumlu mu?" — agent her iki context'i okur, cevap verir.
3. Sentor pencereye bakar, MCP önerir Sentor
Sentor'u canvas'a pencere olarak çağır, bir terminal node'una bağla.
→ Sentor: "Bu cargo check programını agent ile kullanmak için MCP'ye ihtiyaç yok — doğrudan bash_run tool yeterli. Ama cargo clippy warning'lerini structured olarak istersen MCP wrapper yazılabilir. Hangisini istersin?"
→ Kullanıcı seçer. Sentor gerekirse Orkestra'ya plan iletir, gerekirse doğrudan agent_spawn ile çözer.
4. Blueprint'i başka canvas'a taşı Blueprint
research-router blueprint mevcut canvas'ta üretildi, kaydedildi.
→ Yeni canvas aç. Sağ tık → Blueprint İmport → research-router seç.
→ 7 node + 6 edge spawn olur. Sadece Start input'unu yeni canvas'taki bir kaynağa bağla → hazır.
5. Sub canvas → Vault projesi Export
Sub canvas'ta bir araştırma yapıldı, 3 vault dosyası okundu, agent yorumlar üretti.
→ Toolbar → Projeye Çevir butonu.
→ Vault-Exporter agent canvas state'i toplar, HTML üretir, Sentor-Maker stil rulebook'una göre formatlar.
→ "Vault'a Hazır: vault/projects/aralık-araştırma/" bildirimi gelir. Indexer otomatik çalışır.
6. Sentor belirsiz istek → clarify Edge case
Kullanıcı: "şu işi yapan bir agent yap"
→ Sentor: "Hangi 'şu iş'? (1) plan analiz / (2) hata özetleme" — tek tur clarify.
→ Kullanıcı "2" → Sentor spec gösterir → onay → agent_spawn.
7. Blueprint import çakışması Edge case
Import edilen blueprint'in içindeki web-fetcher agent zaten mevcut.
→ Diyalog: [Mevcut'u kullan] · [Yeniden adlandır → web-fetcher-2] · [İptal].
→ Yeniden adlandırılırsa edge'ler yeni id'ye otomatik remap.
8. Recursive builder reddi Güvenlik
Kullanıcı Sentor'a: "İçinde Sentor olan bir agent yap".
→ agent_spawn tools listesinde agent_spawn görür → permission_denied döner.
→ Sentor: "Recursive builder yasak. Orkestra üzerinden tetikleyebilirsin."
9. Orkestra paralel agent (Supervisor) Orkestra
"Bu üç vault sayfasını oku, her birini ayrı agent özetlesin" → Orkestra agent_invoke × 3 paralel.
→ Sonuçlar bir LLM node ile merge edilir → tek cevap.
→ UI'da küçük progress bar: "3 agent çalışıyor (1/3 tamam)".
10. Format validator yakalar Export
Sub canvas'ta inline <script src="https://cdn..."> var → Vault-Exporter reddeder:
"CDN link yasak. /vendor/mermaid.min.js mevcut. Otomatik düzeltsem mi?"
→ Onay → düzelt → tekrar denetle → yaz.
// 07 · sentor protokolü
Sentor nasıl konuşur — sıkı 5 adım.

Belirsizlik → tek tur clarify. Sonra YAML-ish spec → onay → agent_spawn. Asla onaysız spawn yok. Asla recursive builder yok.

// SENTOR SİSTEM PROMPTU (özet)
You are Sentor, Sentor's Agent Builder.

1. Ambiguous? Ask ONE clarifying question.
2. Propose spec (YAML):
     name, role, base, tools[],
     memory, prompt
3. "Onaylıyor musun? (evet/değiştir/iptal)"
4. evet → agent_spawn(spec)
   değiştir → diff + re-show
   iptal → stop
5. "✓ {name} kaydedildi"

REFUSALS
• Never spawn empty tools[]
• Never include agent_spawn in tools
• Never expose vault_write
  unless Sentor-Maker pattern
// TOOL KONTRATI — agent_spawn
type AgentSpawnSpec = {
  name: string;       // kebab, 2-40
  role: string;       // 1 sentence
  prompt: string;     // ≤ 4000 chars
  tools: string[];    // ≥ 1, in registry
  memory: "session" | "ephemeral";
  baseAgentId?: string;
  parentCanvasId?: string;
}

type Result =
  | { ok: true; id: string }
  | { ok: false; error:
      | "name_taken"
      | "invalid_tool"
      | "rate_limit"    // max 10/session
      | "permission_denied" }
// TOOL KONTRATI — blueprint_save
type BlueprintSaveSpec = {
  slug: string;
  name: string;
  description: string;
  selection: {
    nodeIds: string[];   // empty = whole canvas
    edgeIds: string[];
  };
}

type Result =
  | { ok: true; path: string;
      nodeCount: number;
      edgeCount: number }
  | { ok: false; error:
      | "slug_taken"
      | "empty_selection"
      | "invalid_graph" }
// TOOL KONTRATI — canvas_read_state
type CanvasStateSnapshot = {
  canvasId: string;
  parentCanvasId: string | null;
  nodes: Array<{
    id; type; summary; data
  }>;
  edges: Array<{ from; to; kind }>;
  agents: Array<{ id; name; role }>;
  openVaultFiles: string[];
  activeTerminalCwd: string | null;
}
